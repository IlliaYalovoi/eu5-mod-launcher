# 29.4 — Search Popup Not Working for Victoria 3

## Context

Search overlay for Vic3 shows no results when searching by mod name, even though mods exist and are visible in the mod list.

## Expected Behavior

Search should find mods by name or tags, same as EU5.

## Root Cause

The `SearchOverlay.vue` `filteredMods` computed filters `allMods.value` by `mod.Name.toLowerCase().includes(q)`. For Vic3, if `mod.Name` is empty (because Vic3 metadata parsing isn't populating the Name field), all mods would be filtered out by any non-empty query.

The Vic3 mod descriptor format uses `.metadata/metadata.json` with `name` key. The `parseJSONDescriptor` in `descriptor.go` extracts `name` → `Descriptor.Name`, which should flow into `mods.Mod.Name`. Need to verify this chain is working.

## What to Check

1. Do Vic3 mods in `allMods` have non-empty `Name` fields?
2. Is the `descriptor.mod` path resolution working for Vic3 mods? Vic3 uses `.metadata/metadata.json`, not `descriptor.mod`.
3. Add backend logging to `GetAllMods` to verify Name values for Vic3 mods.
