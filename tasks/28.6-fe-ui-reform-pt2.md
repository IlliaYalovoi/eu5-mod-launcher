# Task 28.6 — UI Reform Fixes

## Context

Task 28.5 shipped but has 7 bugs/regressions that need fixing.

---

## Bugs

### 1. Create Category button broken

**Problem**: Category creation via input + button in `LoadOrderPanel` stopped working. No error, no action.

**Root cause**: Likely the `onCreateCategory` function or the input binding broke during refactor.

**Fix**: Verify `categoryName` ref, `onCreateCategory` function, and button click binding in `LoadOrderPanel`.

---

### 2. Load order panel hugs left edge with no breathing room

**Problem**: Load order panel fills the full width with zero margin/padding — feels hyper tight and broken.

**Root cause**: The `.content` grid area in `App.vue` uses the full width with no padding.

**Fix**: Add horizontal padding to the content area:
```css
.content {
  padding: var(--space-5);
}
```
Also ensure `LoadOrderPanel` itself has proper internal spacing.

---

### 3. Game selector sidebar removed

**Problem**: Task 28 (multi-game sidebar) was still pending — the game switcher that was supposed to be in a left sidebar per task 28 spec is gone entirely since `ModListPanel` (which had the playset selector) was removed.

**Fix**: Add a game switcher bar to the header command bar in `App.vue`. The playset selector already exists in `ModListPanel` — move the dropdown into the header, next to the app title. Show current game name, dropdown to switch. Keep it slim in the header, not a sidebar.

---

### 4. Blue accent feels too aggressive

**Problem**: `#3b82f6` blue used everywhere as primary accent feels like every UI element is "clickable" and creates visual noise.

**Fix**: Reduce blue to only interactive affordances that need emphasis (focus rings, selected states, links). Use a more neutral default — revert primary accent back to a neutral gray or the previous gold-toned border color for non-launch buttons. Blue stays only for: focus rings, active selected mod rows, links in text.

Updated accent strategy:
- Primary button: `--color-accent` stays blue (it's a call to action)
- Non-button UI: use `--color-text-muted` / `--color-border` for structure
- Selected/active mod rows: use a subtle left border accent instead of full row blue tint

---

### 5. Settings and mod details z-index conflict

**Problem**: Opening settings (slide-over from right) while mod details is already open causes mod details to overlap settings.

**Fix**: Ensure only one slide-over can be open at a time. When opening settings, close details. When opening details, close settings (unless launched from within settings). Add a `detailsOpen` / `settingsOpen` coordination — opening one closes the other.

---

### 6. Click selected mod should deselect / close details

**Problem**: Clicking the same mod that's already selected doesn't close the details panel — it stays open.

**Fix**: In `onModSelect`, if `modID === selectedModID && detailsOpen`, close details instead.

---

### 7. Category rename inline

**Problem**: No way to rename categories. They should be editable inline, similar to the load order index editing.

**Fix**: In `LoadOrderPanel`, for each category header:
- Show category name as text by default
- On click → replace with an input field (inline edit mode)
- Enter/Escape to confirm/cancel
- Same pattern as `globalEditModID` / `globalEditValue` but for category names

State needed:
```ts
const editingCategoryID = ref('')
const editingCategoryName = ref('')
```
