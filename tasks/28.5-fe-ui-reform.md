# Task 28.5 — Frontend UI Reform

## Context

Current UI is functional but not polished. The 3-column layout is sound, but there are UX debt issues accumulated across tasks. This is a bold reform pass, not incremental polish.

---

## Principle

**Three zones, one job each.** Every panel does ONE thing. Remove dual-purpose components. Reduce cognitive load.

---

## Layout Changes

### Current: 3-column with fixed header
```
[Header bar: title + settings]
[Left: mod list + search] [Center: load order + categories] [Right: mod details]
```

### Proposed: 2-row command center
```
[Header: game switcher + search + actions]           ← Command bar (slim)
[Left: load order (source of truth)] [Right: inspector]  ← 2-column, no left sidebar
```

**Why:**
- Mod discovery (`ModListPanel`) and load ordering were forced into one view → cognitive overload
- Users don't browse mods like a file browser; they think in terms of "what's in my load order"
- Inspector (details) never needs to be the primary focus

**Changes:**
1. **Remove `ModListPanel` as primary view.** It becomes a floating drawer/overlay triggered by "Add mod" or `Ctrl+F` with no results
2. **Load order IS the main view.** Everything starts here
3. **Mod details move to a slide-over panel** (right 30%), not a permanent column
4. **Game switcher moves to header** — not a collapsible left sidebar (per task 28)

---

## Component Changes

### A. LoadOrderPanel — elevate to first-class

**Problems:**
- Category collapse UX is unclear
- Inline index editing is hidden (click number to edit)
- Drag handle is `☰` emoji — looks janky
- Empty categories render as ghost blocks

**Changes:**
- Replace `☰` with proper grip-vertical SVG icon
- Index editing: show as editable badge on hover, not hidden click target
- Empty categories: hide completely or show "Drop mods here" placeholder
- Add `+ Category` button inline, not buried in context menu
- Category colors: assign each category a distinct muted hue from a preset palette

### B. ModListPanel — demote to overlay

**Problems:**
- It's always visible, occupying 30% of screen
- Search exists but filters by name only (no tag, no status)
- Bulk operations missing

**Changes:**
- Convert to a floating search overlay (Cmd/Ctrl+K to open)
- Full-text search across name + tags + workshop title
- Multi-select with checkboxes, bulk enable/disable/add to category
- Close on Escape or click outside

### C. ModDetailsPanel — slide-over inspector

**Problems:**
- Occupies 30% permanently even when user doesn't need it
- Steam description renders HTML in a cramped panel
- Unsubscribe button is visually loud

**Changes:**
- Render as right-side slide-over (40% width) triggered by mod click
- Dismissible by clicking elsewhere or pressing Escape
- Steam description in a scrollable, well-padded area
- Unsubscribe moved to bottom, styled as ghost/danger, requires confirmation modal (not `window.confirm`)

### D. ConstraintModal — simplify

**Problems:**
- `first`/`last` vs `after`/`before` constraint types confuse users
- No visual graph of constraints
- Existing constraint rows don't clearly name both mods

**Changes:**
- Unify constraint types to `after` and `before` only — remove `first`/`last` (can be expressed as "after nothing" edge cases)
- Show a mini dependency graph: nodes are mod names, edges show direction
- Each constraint row: `[Mod A] → [Mod B]` with delete button
- Add button: autocomplete ModPicker, direction toggle, confirm

### E. SettingsPanel — slide-over, not modal

**Problems:**
- Modals interrupt flow
- Path inputs are readonly with no copy button

**Changes:**
- Convert to slide-over panel from right (same as mod details)
- Add copy-to-clipboard icon next to each path
- Add "Reset to defaults" link
- Show detected paths with green checkmark, manual overrides with edit icon

### F. LaunchButton + AutosortButton — status-aware

**Changes:**
- Show full game name being launched: "Launch Crusader Kings III" not just "Launch"
- Autosort: show brief animation (reorder ripple) when sorting happens, not just success flash
- Both buttons in a sticky footer bar with the load order list above

### G. Toast notifications — redesign

**Current:** Bottom-center toast, auto-dismiss 3.2s

**Changes:**
- Stack in bottom-right (not bottom-center — avoids finger/hover zone on laptops)
- Max 3 visible; overflow collapses into "+N more"
- Types: success (green), error (red), info (blue), warning (amber)
- Errors persist until dismissed; success auto-dismisses

### H. CycleErrorPanel — integrate into autosort flow

**Changes:**
- Don't show as a slide-in blocking panel
- After failed autosort, the button stays in error state
- Clicking the error button opens ConstraintModal focused on the problematic mod
- Show cycle path as a small inline breadcrumb: `ModA → ModB → ModC → ModA`

## Visual Design Changes

### Typography
- Keep Cinzel + Nunito + JetBrains Mono
- Reduce Cinzel to titles only (h1, h2); body uses Nunito
- Monospace for paths, mod IDs, technical values

### Spacing
- Standardize on 4px grid
- Panel gaps: 16px
- Component internal padding: 12px / 16px
- Generous whitespace in inspector panel

### Color
- Remove gold accent for primary actions — use a more neutral blue (`#3b82f6`)
- Gold reserved for "launch" / "play" / positive confirmation only
- Dark theme: `--color-bg-base: #0f1117` (not current — evaluate)
- Error states: use semantic red, not border-contrast

### Motion
- Panel slides: 200ms ease-out
- List reorders: 150ms spring
- Drag ghost: 10% opacity, slight scale-up
- No motion for state-only changes (loading spinners OK)

---

## Interactions

### Keyboard shortcuts
| Key | Action |
|---|---|
| `Ctrl+K` | Open mod search overlay |
| `Ctrl+S` | Save (persist load order — even though it auto-saves, gives feedback) |
| `Ctrl+F` | Focus search (in load order) |
| `Escape` | Close any open panel/modal |
| `Delete/Backspace` | Remove selected mod from load order (with confirmation) |
| `Ctrl+Shift+↑/↓` | Move selected mod up/down |

### Drag and drop
- Visual drop indicator: 2px accent-colored line between items (not ghost placeholder)
- Drag handle on left, entire row is a drop target
- Show drop preview before releasing mouse

---

## Accessibility
- All interactive elements: visible focus ring (current: `outline: none` on some inputs)
- Dialogs: trap focus, return focus on close
- Color contrast: audit all text against background tokens
- `aria-label` on icon-only buttons
