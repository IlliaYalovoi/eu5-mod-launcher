# 29.5 — Playset Change Does Not Refresh Load Order

## Context

When changing the active playset via the dropdown in mod list panel, the load order does not refresh — ordered IDs and layout remain the old playset's data.

## Expected Behavior

Changing playset should:
1. Load new playset's `OrderedIDs` and `modPathByID` via `ImportModList`
2. Rebuild `launcherLayout` with `defaultLauncherLayout` using new ordered IDs
3. Frontend `loadOrderStore.fetch()` should pull updated data

## Root Cause

`LoadOrderPanel.vue` `onLauncherPlaysetChange` calls `loadOrderStore.setLauncherPlayset(index)` which calls `SetLauncherActivePlaysetIndex`. The backend `SetLauncherActivePlaysetIndex` updates `a.launcherIdx` but does NOT reload the playset state.

Looking at `app_game.go:146`:
```go
func (a *App) SetLauncherActivePlaysetIndex(idx int) error {
    a.launcherIdx = idx
    return nil  // <-- Returns without reloading playset state!
}
```

It should call `ImportModList` with the new index to reload `a.loadOrder` and `a.modPathByID`.

## What to Fix

`SetLauncherActivePlaysetIndex` must reload playset state after updating the index:
```go
func (a *App) SetLauncherActivePlaysetIndex(idx int) error {
    a.launcherIdx = idx
    state, pathByID, err := a.svc.gameSvc.ImportModList(...)
    if err == nil {
        a.loadOrder = state
        a.modPathByID = pathByID
        a.launcherLayout = defaultLauncherLayout(state.OrderedIDs)
    }
    return nil
}
```

Also verify `SetActiveGame` path already handles this (it does, based on task 29.1 analysis).
