# Task 28.8: Backend Rewrite Plan

## Goal
Rewrite backend to be: minimal, game-agnostic, dependency-injected, AI-consumable.

---

## Problems with Current Code

| Issue | Location |
|---|---|
| `app.go` is a 1400+ line god object | `app.go` |
| Domain types scattered (`domain/types.go`, `game/contracts.go`, `mods/mod.go`) | multiple |
| Services created but not properly injected; logic duplicated in `app.go` | `app.go` |
| No interfaces for repositories (hard to test/mock) | `repo/*` |
| `loadorder.State` is a plain struct, not a value object | `loadorder/store.go` |
| Steam client does too much (fetch, cache, parse, images, descriptions) | `steam/*` |
| `App` struct holds raw fields instead of service references | `app.go` |

---

## Target Architecture

```
internal/
├── domain/           # Pure types, zero dependencies
│   ├── mod.go        # Mod, ModID, CategoryID
│   ├── game.go       # GameID, GameDescriptor, GamePaths
│   ├── constraint.go # Constraint, ConstraintType
│   ├── loadorder.go  # LoadOrder (value object)
│   └── errors.go     # Typed errors
│
├── game/             # Game adapter interfaces + implementations
│   ├── adapter.go    # GameAdapter interface
│   └── eu5/          # EU5 concrete adapter
│
├── repo/             # Repository interfaces (abstractions for data access)
│   ├── settings.go
│   ├── constraints.go
│   ├── loadorder.go
│   └── playset.go
│
├── service/          # Business logic, depends on repos/adapters
│   ├── mods.go
│   ├── loadorder.go
│   ├── constraints.go
│   ├── game.go
│   └── launch.go
│
├── steam/            # Steam web API client (single responsibility)
│   └── client.go
│
└── graph/            # Constraint graph + topological sort (keep as-is, minimal)

cmd/
└── server/
    └── main.go       # Wire everything, create App

app.go                # Thin Wails bridge: delegates to services, returns JSON-friendly types
```

## Breaking Changes to Document

Frontend will break on these Wails method signatures:

| Old Method | New Method | Notes |
|---|---|---|
| `GetGames()` | `DiscoverGames()` | Returns `[]GameDescriptor` |
| `GetMods()` | `DiscoverMods()` | Returns `[]domain.Mod` |
| `GetLoadOrder()` | `GetLoadOrder()` | Returns `LoadOrder` struct |
| `SetLoadOrder([]string)` | `SetLoadOrder(LoadOrder)` | Takes struct, not raw slice |
| `AddConstraint(string, string)` | `AddConstraint(Constraint)` | Takes `Constraint` struct |
| `Autosort()` | `AutoSort()` | Renamed, returns ordered mod IDs |
| Various | | All IDs now typed (`ModID`, `GameID`) |

## Principles

1. **No god objects** — max ~300 lines per file
2. **Interfaces first** — define interface before implementation
3. **Dependency injection** — services receive deps via constructor, not global state
4. **Domain is pure** — `domain/*` has zero imports outside standard library
5. **AI-optimized** — small files, clear names, single responsibility per function
6. **Game-agnostic core** — `GameAdapter` interface makes adding Vic3/HUN/etc trivial
