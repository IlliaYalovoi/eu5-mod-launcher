# 29.3 — EU5 Load Order: Some Mod Names Fall Back to IDs

## Context

In EU5 load order, 3 specific mods (e.g. `3644173888`) display their numeric IDs instead of names. These same mods show correct names in the mod list panel.

## Expected Behavior

All mods in load order should display human-readable names, same as in mod list.

## Root Cause

The `LoadOrderPanel.vue` builds `modsByID` from `allMods`, which has correct names. However, somewhere in the display pipeline, the mod's ID is being shown instead of its name for certain mods.

Likely cause: these 3 mods have IDs that are purely numeric strings (like `3644173888`). When the load order is serialized/deserialized or compared, some code path is treating numeric IDs differently than string IDs, or there's a coercion issue where the mod lookup fails and falls back to displaying the ID.

## What to Investigate

1. Do these mods have special characters in their names or IDs?
2. Is there any coercion between `string` and `number` happening?
3. Does the issue affect mods whose IDs are purely numeric?
