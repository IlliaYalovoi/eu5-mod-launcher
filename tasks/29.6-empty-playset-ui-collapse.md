# 29.6 — Empty Playset Causes UI Collapse

## Context

Switching to a Vic3 playset with 0 mods (0 added, 0 enabled) causes the entire UI to go dark — all panels disappear, leaving only the background color visible. The app doesn't crash (logs continue), but the UI is non-functional.

## Expected Behavior

An empty playset should display an empty load order list and mod list, with a message like "No mods in this playset".

## Root Cause

Likely a division by zero or index out of bounds when `orderedIDs.length === 0`. The `LoadOrderPanel.vue` computed `compiledOrder` iterates over `blocks.value`, and if `blocks` is empty or has invalid state, rendering may fail.

Possibly in `numberByModID` or `localNumberByModID` computed properties, or in the draggable component initialization when passed empty arrays.

## What to Investigate

1. Add defensive checks in `LoadOrderPanel.vue` computed properties for empty arrays.
2. Check if any component expects at least 1 mod and fails gracefully when 0.
3. Verify `defaultLauncherLayout([])` handles empty ordered IDs correctly.
4. Check browser console for JS errors ( Vue error boundary not catching the exception).
