# 29.2 — Vic3 Mod Details: Missing Workshop ID / Steam Enrichment Broken

## Context

When viewing mod details for a Victoria 3 mod, the Steam metadata panel shows an error: `no workshop id for mod "3602694948" Retry`. Thumbnails also don't load.

## Expected Behavior

Vic3 workshop mods (subscribed via Steam workshop) should fetch Steam workshop metadata and display thumbnails, same as EU5 mods.

## Root Cause

The `steamItemIDForMod` and `workshopItemIDFromDirPath` functions in `mods.ts` extract the Steam workshop ID from the mod's `DirPath` by matching against `steamAppID = "3450310"` (EU5). Vic3 uses app ID `529340`, so workshop mod paths under `workshop/content/529340/` don't match.

```typescript
// mods.ts:64 - hardcoded EU5 app ID
const steamContentPattern = new RegExp(`/workshop/content/${steamAppID}/(\\d+)(?:/|$)`, 'i')
```

## What to Fix

1. The `steamAppID` constant in `mods.ts` is hardcoded to EU5's app ID.
2. Game-specific Steam app ID should come from the active game's configuration, not a global constant.
3. The `GetAllMods` backend could pass game info that helps the frontend resolve the correct app ID.
