.# Task 28.1: Debug Logging Infrastructure

## Problem Statement
Currently there's no structured logging in frontend or backend, making troubleshooting difficult. Need to add comprehensive logging to both frontend and backend.

## Frontend Logging Requirements
- Use console with structured log levels: debug, info, warn, error
- Create a logging module at `frontend/src/lib/logger.ts`
- Configure different log levels for dev/prod (debug in dev, error-only in prod)
- Log: store operations, API calls, state changes, user interactions

## Backend Logging Requirements
- Backend already has `internal/logging` package - audit what's there
- Add structured logging to:
  - Game detection service (games found, paths detected)
  - Mods service (mod loading, filtering)
  - Load order service (operations, saves)
  - Wails method handlers (incoming requests, responses)
- Use levels: DEBUG, INFO, WARN, ERROR
- Log format: `[LEVEL] timestamp component: message {structured data}`

## Acceptance Criteria
- [ ] Frontend has logger module with level-based logging
- [ ] Frontend logs store initialization and state changes
- [ ] Backend logs game detection results
- [ ] Backend logs mod loading/filtering operations
- [ ] Logs are helpful for debugging game switching issues
- [ ] No excessive logging that clutters console
